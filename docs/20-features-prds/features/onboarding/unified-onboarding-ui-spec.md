## Data Architecture

### Field Definitions & Storage

All three input modes (voice, chat, form) write to the **same data stores**:

| Field | Storage Location | Level | Description |
|-------|------------------|-------|-------------|
| `website_url` | `accounts.accounts` | Account | Company website for auto-research |
| `company_description` | `accounts.accounts` | Account | What the company does |
| `customer_problem` | `accounts.accounts` | Account | Pain point solved |
| `offerings` | `accounts.accounts` | Account | Products/services offered |
| `target_orgs` | `accounts.accounts` | Account | Industry/org types served |
| `target_roles` | `accounts.accounts` | Account | Job titles of buyers/users |
| `competitors` | `accounts.accounts` | Account | Known competitors |
| `industry` | `accounts.accounts` | Account | Company's industry |
| `research_goal` | `project_sections` | Project | What this research aims to learn |
| `assumptions` | `project_sections` | Project | Hypotheses to test |
| `unknowns` | `project_sections` | Project | Questions to explore |
| `decision_questions` | `project_sections` | Project | Generated interview prompts |
| `target_roles` (override) | `project_sections` | Project | Project-specific role overrides |
| `target_orgs` (override) | `project_sections` | Project | Project-specific org overrides |

### ProjectSetupProvider Shape

```typescript
interface ProjectSetupState {
  // Account-level fields
  account: {
    website_url: string | null;
    company_description: string | null;
    customer_problem: string | null;
    offerings: string[];
    target_orgs: string[];
    target_roles: string[];
    competitors: string[];
    industry: string | null;
  };

  // Project-level fields
  project: {
    research_goal: string | null;
    assumptions: string[];
    unknowns: string[];
    decision_questions: string[];
    target_roles_override: string[] | null;
    target_orgs_override: string[] | null;
  };

  // Extraction metadata (for traceability)
  extractions: {
    fieldKey: string;
    value: string | string[];
    sourceMessageId: string;
    sourceText: string;
    extractedAt: string;
  }[];

  // UI state
  mode: 'chat' | 'voice' | 'form';
  capturedPaneExpanded: boolean;

  // Actions
  setAccountField: (key: string, value: unknown) => void;
  setProjectField: (key: string, value: unknown) => void;
  setMode: (mode: 'chat' | 'voice' | 'form') => void;
  toggleCapturedPane: () => void;
}
```

Contextual AI-powered suggestions are **essential** for all input modes. They:

1. **Reduce cognitive load** - Users don't stare at blank fields

### Behavior

| Action | Result |
|--------|--------|
| Click `+ suggestion` | Adds suggestion to field (or list if tags/array field) |
| Click `Ã—` next to suggestion | Rejects it, won't show again, AI learns |
| Click `[â†» Refresh]` | Generates new suggestions based on context |
| Type manually | Suggestions remain visible as alternatives |

### Required for All Field Types

| Field Type | Suggestion Behavior |
|------------|---------------------|
| `textarea` | Suggestions appear below input, clicking populates field |
| `tags` | Clicking adds tag to list, suggestion disappears from options |
| `url` | No suggestions (user-specific data) |
