## User Flows

### Stakeholder Consultation Flow
```
1. CAPTURE
   └─ Structure stakeholder data
   └─ Customize fields per project
   └─ Import existing contacts
   └─ Segment by project/phase/site

2. TRACK INTERACTIONS
   └─ Log meetings, calls, emails
   └─ Record sentiment
   └─ Tag issues and topics
   └─ Assign actions

3. ANALYSE
   └─ Run reports on engagement
   └─ View interrelationships
   └─ Identify patterns (manual)
   └─ Track resolution status

4. ENGAGE
   └─ Multi-channel outreach
   └─ Surveys and feedback
   └─ Newsletter campaigns
   └─ Track responses
```

## Competitive Position

### Their Strengths

1. **Enterprise Security**: ISO 27001, SOC 2, Cyber Essentials
2. **Customization**: Adapt to any engagement process
3. **UK Public Sector**: Strong government relationships
4. **Multi-Industry**: Energy, healthcare, education, infrastructure
5. **Compliance Focus**: Built for regulated environments

### Their Weaknesses

1. **No Voice**: Cannot capture or transcribe interviews
2. **No AI**: All analysis is manual
3. **Data Entry Heavy**: Everything requires manual input
4. **No Synthesis**: Cross-stakeholder patterns are manual
5. **No Deliverables**: Doesn't generate client outputs
6. **Opaque Pricing**: Must contact sales

## Threat Assessment

**Threat Level**: LOW

**Why Low**:
- Different market (infrastructure/government vs. consulting)
- No AI/voice direction visible
- Enterprise-only positioning
- UK-focused market

**Risk Scenario**:
- If they add AI/voice and target consulting, could compete

**Watch For**:
- AI feature announcements
- Voice interview features
- Consulting-specific marketing
