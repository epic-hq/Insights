## User Flows

### Stakeholder Engagement Flow
```
1. CREATE STAKEHOLDER
   └─ Add contact details
   └─ Assign to project/location
   └─ Set stakeholder type/category

2. LOG INTERACTIONS
   └─ Record meeting/call/email
   └─ Attach notes
   └─ Tag issues raised
   └─ Assign follow-up actions

3. MANAGE ISSUES
   └─ Create issue from interaction
   └─ Assign priority/owner
   └─ Track resolution status
   └─ Link to commitments made

4. REPORT & COMPLY
   └─ Generate engagement reports
   └─ Export for regulatory submission
   └─ Dashboard views by project/region
```

### Key Workflow Difference from UpSight
- **Darzin**: Manual data entry, tracking-focused
- **UpSight Opportunity**: Automatic capture + AI analysis + deliverables

## Competitive Position

### Their Strengths

1. **Longevity**: 20 years in market, trusted brand
2. **Specialization**: Purpose-built for infrastructure/resources
3. **Compliance**: Designed for regulatory requirements
4. **Government**: Strong public sector presence
5. **Geographic mapping**: Spatial stakeholder views

### Their Weaknesses

1. **No AI**: Entirely manual analysis
2. **No Voice**: Can't capture/transcribe interviews
3. **Data Entry Heavy**: Requires manual logging
4. **No Synthesis**: Cross-interview analysis is manual
5. **No Deliverables**: Doesn't generate client outputs
6. **Enterprise Pricing**: Not accessible to SMB consultants

## Threat Assessment

**Threat Level**: LOW-MEDIUM

**Why Low**:
- Different market focus (infrastructure vs. consulting)
- Enterprise pricing excludes our SMB target
- Manual-first approach is opposite of our value prop

**Risk Scenario**:
- If they add AI/voice features, could compete for larger consulting firms

**Watch For**:
- AI feature announcements
- Voice/interview capture additions
- SMB pricing tier launch
