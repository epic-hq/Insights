Interaction States & Behavior

! New message arrives while reading

When user sends a new message while TTS is playing:

  1. 1. Stop current playback immediately
  2. 2. Cancel any queued audio chunks
  3. 3. Wait for new response to start streaming
  4. 4. Begin reading new response (if toggle still on)

» Streaming TTS (sentence chunking)

Don't wait for full response. Stream audio by sentence:

  1. 1. Buffer text until sentence boundary (. ! ? or newline)
  2. 2. Send sentence to OpenAI TTS API (streaming endpoint)
  3. 3. Play audio chunk while buffering next sentence
  4. 4. Overlap: fetch chunk N+1 while playing chunk N

Perceived latency: ~300ms from first sentence complete to audio start

S Session persistence

Toggle state persists via:

  • - localStorage key: uppy-tts-enabled
  • - Survives page refresh and new chat sessions
  • - Does NOT auto-play past messages on load
  • - Only triggers on NEW assistant responses

V Voice chat coexistence

When LiveKit voice chat is active:

  • - Auto-disable TTS toggle (LiveKit handles audio)
  • - Show tooltip: "TTS disabled during voice call"
  • - Re-enable when call ends (restore previous state)
  • - Per-message play buttons hidden during call
